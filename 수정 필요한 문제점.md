# 수정 필요한 문제점

## 1. 대용량 파일 이관 시 SPS 블록 이동 중단 현상 (3개 블록에서 멈춤)

### 문제 현상
- `~/test-auto-tiering-savings.sh A` 스크립트 실행 등 대용량 파일(1GB 이상)을 이관할 때, 초반 3개의 블록만 전송된 후 더 이상 블록 이동이 진행되지 않고 영원히 `IN_PROGRESS` 상태에 머무는 현상 발생.

### 원인
HDFS 코어 설정(`hdfs-site.xml`)의 기본 제약으로 인한 SPS(Storage Policy Satisfier) 데몬의 이관 포기(Drop) 동작.
1. **NameNode 동시 전송 한도 제한**: NameNode는 `dfs.namenode.replication.max-streams`의 기본값이 2로 설정되어 있어 한 번에 최대 2개의 블록 전송만 허용함.
2. **SPS 데몬의 재시도 대기 시간**: 거절당한 블록들을 재요청할 때 SPS 데몬은 `dfs.storage.policy.satisfier.recheck.timeout.millis`의 기본값인 1분(60000ms)을 대기함.
3. **최대 재시도 횟수 제한**: SPS 데몬은 `dfs.storage.policy.satisfier.retry.max.attempts`의 기본값이 3으로 설정되어 있어, 1분씩 3번 재시도한 후에도 전체 블록을 옮기지 못하면 해당 파일의 이관을 영구적으로 포기(대기열에서 삭제)함.

### 해결 방안 (수정 내역)
`~/hadoop-conf/namenode/hdfs-site.xml` 설정 파일에 아래의 튜닝 파라미터를 추가하여 제약을 해제하고, NameNode 및 SPS 데몬을 재시작해야 함.

```xml
  <!-- SPS 재시도 횟수 제한 해제 (기본 3 -> 100) : 3번 실패 시 포기 방지 -->
  <property>
    <name>dfs.storage.policy.satisfier.retry.max.attempts</name>
    <value>100</value>
  </property>

  <!-- SPS 대기 시간 단축 (기본 60000ms -> 3000ms) : 1분 대기 방지 -->
  <property>
    <name>dfs.storage.policy.satisfier.recheck.timeout.millis</name>
    <value>3000</value>
  </property>

  <!-- NameNode 동시 블록 이동 허용량 대폭 증가 (기본 2 -> 20) -->
  <property>
    <name>dfs.namenode.replication.max-streams</name>
    <value>20</value>
  </property>

  <!-- NameNode 동시 블록 이동 하드 리미트 증가 (기본 4 -> 40) -->
  <property>
    <name>dfs.namenode.replication.max-streams-hard-limit</name>
    <value>40</value>
  </property>
```
